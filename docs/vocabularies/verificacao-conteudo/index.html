Resumo

O campo Verificação de Conteúdo é usado para indicar a licença sob a qual um recurso de dados é disponibilizado em catálogos de dados públicos no Brasil. Este vocabulário controlado baseia-se no padrão SPDX License List mantido pela SPDX (Software Package Data Exchange), uma iniciativa da Linux Foundation que padroniza a identificação de licenças de software e conteúdo.

Status deste documento

Este documento descreve o vocabulário controlado para Verificação de Conteúdo mantido pelo Grupo de Trabalho DCAT-BR. Comentários e propostas de alteração devem ser enviados por meio do repositório público indicado na seção de governança.

Introdução

O campo Verificação de Conteúdo é uma propriedade essencial no perfil DCAT-BR para descrever a licença sob a qual uma distribuição de dados está disponível. Este campo utiliza a propriedade dct:license do vocabulário Dublin Core Terms e deve referenciar um identificador de licença do padrão SPDX.

O padrão SPDX fornece uma lista padronizada de identificadores de licenças amplamente utilizadas em projetos de software e dados abertos. O uso deste vocabulário garante interoperabilidade e facilita a identificação clara dos termos de uso dos dados disponibilizados.

Selo de Conformidade: Ouro
Base: SPDX License List
URI Base: https://spdx.org/licenses/

Uso do Vocabulário

O campo Verificação de Conteúdo deve referenciar identificadores de licença do padrão SPDX. Os valores devem ser expressos como URIs que identificam a licença específica. Alguns exemplos comuns de licenças utilizadas em dados abertos incluem:

Identificador SPDX Nome da Licença Descrição
CC0-1.0 Creative Commons Zero v1.0 Universal Dedicado ao domínio público, sem restrições de uso
CC-BY-4.0 Creative Commons Attribution 4.0 Permite uso comercial e não comercial, requer atribuição
CC-BY-SA-4.0 Creative Commons Attribution-ShareAlike 4.0 Permite uso, requer atribuição e compartilhamento sob mesma licença
ODbL-1.0 Open Database License Licença específica para bancos de dados abertos
ODC-BY-1.0 Open Data Commons Attribution License Licença para dados abertos com requisito de atribuição
PDDL-1.0 Public Domain Dedication and License Dedicação ao domínio público para dados
OGL-3.0 Open Government Licence Licença governamental aberta (Reino Unido)
NLOD-1.0 Norwegian Licence for Open Government Data Licença norueguesa para dados governamentais abertos

Para uma lista completa de licenças SPDX, consulte o registro oficial do SPDX. Cada licença possui um identificador único e uma URI correspondente que deve ser utilizada no campo dct:license.

Exemplo de Uso

Abaixo está um exemplo de como usar o campo Verificação de Conteúdo (licença) em uma distribuição DCAT-BR: 

@prefix dcat: <http://www.w3.org/ns/dcat#> .
@prefix dct: <http://purl.org/dc/terms/> .

<https://exemplo.gov.br/dataset/1/distribution/1>
    a dcat:Distribution ;
    dct:title "Dados em CSV"@pt-BR ;
    dct:license <https://spdx.org/licenses/CC-BY-4.0.html> ;
    dcat:downloadURL <https://exemplo.gov.br/dados.csv> .

Alternativamente, pode-se usar o identificador SPDX diretamente: 

@prefix dcat: <http://www.w3.org/ns/dcat#> .
@prefix dct: <http://purl.org/dc/terms/> .

<https://exemplo.gov.br/dataset/1/distribution/1>
    a dcat:Distribution ;
    dct:title "Dados em CSV"@pt-BR ;
    dct:license <https://spdx.org/licenses/CC-BY-4.0.html> ;
    dcat:downloadURL <https://exemplo.gov.br/dados.csv> .
